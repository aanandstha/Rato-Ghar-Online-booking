# SRS
1. User Personas (Primary Users)The brief asks: Who are the primary users? For Rato Ghar, you need at least two distinct personas.Persona NameRoleGoalsActions to PerformRohan (The Student)CustomerWants a quick, secure dinner order between classes.Browses menu, adds to cart, pays via Stripe, tracks order status.Srijana (The Manager)Admin/StaffNeeds to manage the kitchen workflow and update the menu.Views new orders, updates order status, edits item prices/availability.2. User Stories (Actions & Goals)The brief asks: What are their goals? These stories capture the "Why" behind the features.US1: As a Customer, I want to browse the menu by category (e.g., Momos, Curry) so that I can find my favorite food quickly.US2: As a Customer, I want to pay for my order using a secure credit card gateway so that I don't need to use cash.US3: As a Staff Member, I want to receive real-time order notifications so that I can start food preparation immediately.US4: As an Admin, I want to securely log in to a dashboard so that I can manage the menu prices and view sales reports.3. Core Product Features (Functional Requirements)Based on your proposal, these are the "Must-Have" features:User Authentication: Secure registration and login for customers and staff.Digital Menu: Dynamic display of food items from the PostgreSQL database.Cart & Checkout: Real-time price calculation and delivery/pickup selection.Order Status Dashboard: A kitchen-facing view to update orders (Pending -> Preparing -> Completed).4. System Requirements (The "How")Since you are also responsible for System Architecture, you must define the requirements that the technology must meet.Functional RequirementsThe system shall allow users to filter items by dietary preferences (e.g., Vegan).The system shall send a transaction confirmation to the user upon successful payment.Non-Functional Requirements (Critical for Cyber Security)Security: All user passwords must be hashed using Bcrypt algorithms before being stored in the database.Scalability: The system architecture must support up to 100 concurrent users without performance degradation.Privacy: No raw credit card data shall be stored on the Rato Ghar servers; all payments must use Stripe tokenization.
